One home. Many knowledge shapes.

Alcove separates product behavior from user data. Global state lives in Alcove Home; managed knowledge bases live wherever the user chooses.

Managed

Knowledge bases

Inbox review, OKF notes, source archives, validation, gardening, and KB-scoped workflows for content you actively manage.

Global

Pins, tasks, prompts

Small durable items that should be available from any project: references, follow-ups, reusable instructions, operational reminders, and exported mirrors.

Indexed

Mounts and connectors

Read-oriented external memory for local folders, GitHub stars, browser bookmarks, Apple Notes, and future data sources.

Designed around boundaries, not a single app.

Alcove is intentionally split into storage contracts, entry profiles, capture adapters, search surfaces, and export paths.

HomeGlobal registries, indexes, pins, tasks, prompts, usage snapshots.
KB rootUser-chosen managed folders with inbox, archive, and OKF notes.
WorkspaceHub-level control or lightweight business context with local OKF notes.
AdapterCapture tools write bundles into inboxes without owning the knowledge model.
AgentCLI and MCP expose the same operations to Codex, Claude Code, and local scripts.
Capture

Collector-agnostic inboxes

Clipsmith is the default capture adapter, but Alcove only requires the inbox bundle contract.

Search

Home-wide lookup

Managed KBs, global memory, mounts, and connectors participate in one search surface.

Operate

Agent-ready profiles

Install thin project files and skills for hub control, lightweight workspaces, global MCP access, or KB-local work.

Recover

Exportable local state

Global data, managed knowledge bases, and module views can be exported or published outside the runtime.

The interface is small because the contract is explicit.

The same commands work from a hub workspace, an unrelated project, or inside a managed knowledge base.

Alcove session CLI · MCP · local files 
# initialize global state and register a managed KB
$ alcove home init
$ alcove kb add research_notes /path/to/research_notes

# use a local hub for daily personal knowledge work
$ alcove hub init ~/AlcoveHub --default-kb research_notes

# create a lightweight business workspace
$ alcove workspace init family --context "Family planning"

# make memory available through MCP and enable the background Notes mirror
$ alcove global install
$ alcove search "OKF" --home ~/.alcove --json

Workspaces keep intent local.

The Hub remains the special control room for the whole system. Ordinary workspaces stay lightweight: enough guidance for Codex or Claude Code to manage a business context, without exposing every admin workflow.

Hub

Total-system control

Use the Hub for cross-module operations: knowledge base management, connectors, radars, dashboards, publishing, validation, and maintenance.

Workspace

Business context

Create family, work, travel, or research spaces with local OKF notes and focused agent instructions that keep the conversation grounded.

Managed KB

Full inbox workflow

Install KB-local commands when a folder owns capture review, source archiving, note synthesis, validation, and cleanup.

Start local. Stay portable.

Install the CLI, create Alcove Home, then choose the entry profile that matches how you work. 

uv tool install git+https://github.com/OctopusGarage/alcove.git
alcove home init
alcove global install
alcove workspace init family
alcove dashboard --home ~/.alcove build
alcove serve --dashboard --home ~/.alcove --port 8765
